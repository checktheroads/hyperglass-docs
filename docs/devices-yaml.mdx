## credentials

The `credentials` object contains a list of credentials:

<div className={styles.wrapnormal}>

| Field                                      | Description                                                                                      |
| ------------------------------------------ | ------------------------------------------------------------------------------------------------ |
| <FieldName name="username" type="string"/> | SSH Username. If using FRRouting/BIRD, the username can be any arbitrary value (it is not used). |
| <FieldName name="password" type="string"/> | SSH Password if using Netmiko/SSH, or API Key if using FRRouting/BIRD.                           |

</div>

#### Example

```yaml
credentials:
  - credential: &credential1
      username: "user1"
      password: "secret1"
  - credential: &credential2
      username: "user2"
      password: "secret2"
```

## proxies

Each proxy object stores the connection parameters for an SSH proxy.

When a proxy server is defined on a device, hyperglass will first initiate an SSH connection to the proxy, and then initiate a second connection to the target device from the proxy server. This can be helpful if you want to secure access to your routers.

| Field                                            | Default                                         | Description                                                                                                     |
| ------------------------------------------------ | ----------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| <FieldName name="name" type="string"/>           | <FieldDefault value="None"/>                    | Name of the proxy server. Used internally to identify the proxy device.                                         |
| <FieldName name="address" type="string"/>        | <FieldDefault value="None"/>                    | IP Address or hostname of the proxy server.                                                                     |
| <FieldName name="port" type="integer"/>          | <FieldDefault type="integer" value="22"/>       | TCP port of the proxy server's SSH service.                                                                     |
| <FieldName name="credential" type="credential"/> | <FieldDefault value="None"/>                    | [Credential](#credentials) object used to authenticate with the proxy server.                                   |
| <FieldName name="nos" type="string"/>            | <FieldDefault type="string" value="linux_ssh"/> | Network Operating System of the proxy server (Netmiko naming scheme). Currently, only `linux_ssh` is supported. |

#### Example

```yaml
proxies:
    - proxy: &proxy1
        name: "server01"
        address: "10.11.6.204"
        port: 22
        credential: *cred_az01_ipam01
        nos: "linux_ssh"
```

## networks

Each network object defines a network (usually mapped to an ASN), which is used in the hyperglass GUI to group routers. A network object must be associated with a device.

<div className={styles.wrapnormal}>

| Field                                          | Description                                                                 |
| ---------------------------------------------- | --------------------------------------------------------------------------- |
| <FieldName name="name" type="string"/>         | Name of the network. Used internally to identify and reference the network. |
| <FieldName name="display_name" type="string"/> | Name of the network that is shown in the hyperglass GUI.                    |

</div>

#### Example

```yaml
networks:
  - network: &net_primary
      name: "primary"
      display_name: "Main Network"
  - network: &net_secondary
      name: "secondary"
      display_name: "That Other Network"
```
